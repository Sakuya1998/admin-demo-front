# Admin Demo UI 设计规范 (Cyber-Industrial / Hard Sci-Fi Style)

本文档旨在为 Admin Demo 前端项目提供**硬核科幻、工业未来感**的 UI/UX 设计指导。
我们拒绝现代 AI 产品那种“温暖、柔和、拟人化”的视觉语言，转而拥抱**冷峻、精密、机械化**的未来美学。风格参考：*FUI (Fictional User Interface)*、*Cyberpunk*、*Industrial Sci-Fi*。

## 1. 设计系统 (Design System)

### 1.1 色彩体系 (Color Palette)

全站强制 **暗黑模式 (Dark Mode)**。色彩不仅是装饰，更是信号灯和能量指示。

*   **背景 (Void)**:
    *   **全局背景**: `#050505` (Deep Space) - 接近纯黑，但不死黑。
    *   **面板背景**: `#0A0A0A` (Carbon) - 叠加微弱的网格纹理。
*   **能量色 (Energy - Neon & Glowing)**:
    *   **主色 (Primary / System Normal)**: `#00F0FF` (Cyber Cyan) - 用于核心操作、激活状态、数据正常。
    *   **次色 (Secondary / Standby)**: `#7000FF` (Electric Purple) - 用于辅助信息、特殊标记。
    *   **高亮 (Highlight)**: `#FFFFFF` - 仅用于极少数需要刺眼强调的文字。
*   **状态色 (Status)**:
    *   **成功 (Optimal)**: `#00FF9D` (Neon Green)
    *   **警告 (Caution)**: `#FFD600` (Industrial Yellow)
    *   **严重/错误 (Critical)**: `#FF003C` (Crimson Red)
*   **结构色 (Structure)**:
    *   **边框**: `rgba(0, 240, 255, 0.3)` - 细微的发光边框。
    *   **分割线**: `#1A1A1A` - 深灰色工业线条。

### 1.2 字体排版 (Typography)

排版应像**机器终端**输出的数据，强调信息密度和可读性，而非文学性。

*   **字体家族**:
    *   **数据/代码/标题**: `'Orbitron', 'Rajdhani', 'JetBrains Mono', monospace` - 具有科技感或等宽特性的字体。
    *   **界面文本**: `'Inter', 'Roboto', sans-serif` - 保持清晰，但字间距 (Letter-spacing) 稍宽。
*   **排版特征**:
    *   **全大写 (UPPERCASE)**: 模块标题、按钮文字、标签通常使用全大写，增加严肃感。
    *   **字间距**: 标题字间距增加 `+1px` 或 `+2px`。
    *   **字号**:
        *   **H1 / 巨型指标**: 36px, Light/Thin, Monospace.
        *   **H2 / 模块头**: 20px, Bold, Uppercase.
        *   **Body**: 14px, Regular, 颜色偏灰 (`#888`)，高亮词汇用主色。

### 1.3 形状与边框 (Shapes & Borders) - **拒绝圆角**

*   **形状 (Form)**:
    *   **切角 (Chamfered)**: 核心容器使用 `45度` 切角，代替圆角。
    *   **直角 (Sharp)**: 次级元素使用绝对直角。
    *   **装饰性边角 (Corner Accents)**: 容器的四角可能有额外的加粗短线（L型括号）装饰。
*   **边框 (Borders)**:
    *   **极细线**: `1px` 实线，配合微弱的 `box-shadow` 发光效果。
    *   **虚线/刻度**: 使用虚线边框模拟工业图纸或测量刻度。

### 1.4 视觉质感 (Texture & Effects)

*   **发光 (Glow)**: 关键元素（按钮、激活项）带有同色系外发光 `box-shadow: 0 0 8px rgba(0, 240, 255, 0.5)`。
*   **网格 (Grid)**: 背景叠加低透明度的精细网格 (`background-image: linear-gradient(...)`)。
*   **扫描线 (Scanline)**: 某些面板可叠加微弱的横向扫描线纹理。
*   **磨砂玻璃 (Glass)**: 仅在模态窗口背景使用，但要是**深色磨砂**，且带有噪点 (Noise)。

## 2. 整体布局 (Global Layout)

### 2.1 结构：HUD (Heads-Up Display) 风格

布局不再是传统的“网页”，而是飞船或机甲的**驾驶舱控制台**。

1.  **侧边栏 (Sidebar)**:
    *   **外观**: 左侧固定，带有明显的右侧边框（发光）。
    *   **菜单**: 像物理开关或控制杆。选中状态下，左侧亮起光条，背景微亮。
    *   **底部**: 显示系统状态（如：CPU 温度、Ping 值），用小的进度条或数字跳动展示。
2.  **顶部导航 (Header)**:
    *   **外观**: 顶部悬浮条，可能有顶部切角设计。
    *   **面包屑**: 形式如 `SYSTEM > USERS > LIST`，使用 `>` 连接，发光文字。
    *   **右侧**: 时间显示精确到秒（等宽字体），像数字时钟。
3.  **内容区域 (Main Content)**:
    *   **边框**: 每个大的内容模块（Card）都有独立的边框和切角。
    *   **留白**: 模块间距较小，追求紧凑的控制台感。

## 3. 核心组件设计 (Component Specs)

### 3.1 按钮 (Button)

*   **外形**: 矩形或切角矩形。
*   **Primary**:
    *   背景: 半透明主色 `rgba(0, 240, 255, 0.1)`。
    *   边框: `1px solid #00F0FF`。
    *   文字: `#00F0FF`，发光。
    *   Hover: 背景变为 `#00F0FF`，文字变为黑色 (Black)，产生强烈反差（“过载”效果）。
*   **Danger**:
    *   红色边框，红色文字。Hover 时背景变红，文字变黑，甚至出现 Glitch (故障) 抖动效果。

### 3.2 表格 (Data Grid)

*   **样式**:
    *   **线框模式**: 仅保留水平分割线，极细 (`0.5px`)，深青色。
    *   **表头**: 全大写，底部有加粗发光线。
    *   **数据**: 数字使用等宽字体，右对齐。
    *   **行Hover**: 鼠标经过行时，行背景高亮为低透明度白色，且左侧出现一个小三角指示标。

### 3.3 输入框 (Input)

*   **样式**:
    *   背景: 透明或极深灰。
    *   底部边框: `1px solid #333`。
    *   Focus: 底部边框变为 **主色** 且发光，宽度从中心向两边展开动画。
    *   Label: 输入时上浮，变成微小的技术说明文字。

### 3.4 弹窗 (Modal / Alert)

*   **外观**:
    *   居中悬浮的“全息投影”面板。
    *   边框：双线边框。
    *   背景：深色半透明 (`rgba(10, 10, 10, 0.95)`) + 背景模糊 (`backdrop-filter: blur(10px)`).
    *   标题栏：带有条形码装饰或随机生成的序列号。

## 4. 交互动效 (Motion & Feedback)

拒绝平滑的缓动 (Ease-in-out)，追求**瞬间响应**和**机械顿挫感**。

*   **Hover**: 瞬间变色 (`transition: all 0.1s linear`)，不要拖泥带水。
*   **加载 (Loading)**:
    *   **解码效果**: 文字出现时，先显示乱码，然后快速跳动变成正确文字。
    *   **进度条**: 分段式进度条（像电池格），一格格亮起。
*   **点击**: 带有清脆的视觉反馈（如闪烁一次）。

## 5. 辅助功能与细节

*   **装饰性元素**:
    *   在空白处添加技术装饰：细小的 `+` 号定位点、虚线圈、不可交互的十六进制代码。
*   **图表**:
    *   折线图：线条细且发光，下方有渐变填充。
    *   饼图：空心环形图，带有刻度线。

> **Design Mantra**: "Form follows Data. Function is Beauty. Silence is Void."
